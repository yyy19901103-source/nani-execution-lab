製造業のDXは「システム導入」ではなく、**現場を動かす仕組みの再設計**だ。  
本稿では、製造現場で実際に推進してきた経験をもとに、DX全体を17領域に整理したフレームワークを公開する。

> **読み方：** 各領域は独立しているが、底部の「データ・デジタル基盤」が全テーマを支える土台。基盤なしに上物を建てても崩れる。

## 1. AIで管理されたモノづくり

**目標：技能レス・省人化・自律的な品質確保**

製造DXの中核。人の経験・判断に依存していた工程をデジタルに置き換える。

### 1-1. 現場技術・方案の自動提案

| テーマ | 内容 |
|--------|------|
| Spec読取り・割付 | 製造仕様書からAIが要求事項・リスクワードを自動抽出。100点満点での要求充足率を可視化 |
| 検査内容支援 | 工作要求に対するデジタルデータ支援。検査項目・基準値の自動提案 |
| ナレッジ蓄積 | 不適合事象をデータベース化し、傾向分析・再発防止策を自動サジェスト |
| 設計変更影響分析 | 設計変更が製造工程・コスト・品質に与える影響を自動分析 |
| 最適工程生成 | 過去実績を学習し、加工条件・工程順序をAIが自動提案 |

**ポイント：** 「システムが認識できる形での情報整備」が先決。非構造化ドキュメント（PDF・紙）が残っている限り、AIは動かない。

### 1-2. 工作要求の100%文書化

- 作業要領書の作成・蓄積
- サプライヤ技術監査資料の100%ドキュメント化
- 方案ナレッジの構造化蓄積
- マテリアルリスト・工作要求のデジタル化

**KPI目安：** 文書化率100%達成なしにBOP自動生成は不可。まず文書化から。

### 1-3. BOP 100%自動作成

BOP（Bill of Process）は製造の設計図。これが自動生成できれば、工程設計のリードタイムが劇的に短縮する。

```
設計情報（3D CAD + BOM）
        ↓
    AI解析エンジン
        ↓
BOP自動ドラフト生成
（工程・設備・ツール・時間・検査基準）
        ↓
エンジニアレビュー（差分確認のみ）
        ↓
変更・改版の自動反映
```

### 1-4. 小日程管理の強化

現場の日程管理は「中日程→小日程への手動展開」がボトルネックになりやすい。

- 中日程から小日程への**自動展開**
- 小日程調整条件の**形式知化**（ルール化）
- 現場進捗の**リアルタイム取得**（MESとの連携）
- 設備・人員・資材の制約を考慮した**AI自律スケジューリング**

### 1-5. MES（製造実行システム）導入

| フェーズ | 内容 |
|---------|------|
| Phase 1 | MES導入・工程進捗のデジタル管理 |
| Phase 2 | MES-ERPリアルタイム連携、自動実績収集 |
| Phase 3 | 工程完了の自動判定・次工程自動起動、差異の自律分析 |

## 2. XR技術による作業支援

**目標：紙の作業指示ゼロ・遠隔専門家支援の標準化**

### AR/VR 適用ロードマップ

```
Stage 1：教育・訓練への適用
├── 作業者教育・技能訓練VR
├── 設備・プロセス設計レビューVR
└── 危険作業の仮想体験・安全教育VR

Stage 2：現場作業への適用
├── 組立ガイダンスARの展開
├── 遠隔支援AR（専門家とのリアルタイム共有）
└── 検査作業へのAR適用

Stage 3：全工程標準化
├── ペーパーレス作業指示システム
├── AI作業ナビゲーション（次作業の自動提示）
└── AR活用の標準化・全工程展開
```

**実装のポイント：** ARは「既存の紙マニュアルのデジタル化」ではなく「判断ナビゲーション」として設計すること。「次に何をすべきか」をリアルタイムで示すシステムが真価を発揮する。

## 3. コスト可視化・最適化

製造現場のコストは「見えないもの」が多い。DXで可視化する。

| コスト種別 | 可視化内容 |
|-----------|-----------|
| 不良・手戻りコスト | 工程別・品目別の不良発生コストを自動集計 |
| 品質コスト削減効果 | 改善施策ごとの削減額を自動算出・ROI提示 |
| エネルギーコスト | 設備別・工程別の電力・ガス・水消費を見える化 |
| カーボンフットプリント | Scope3排出量をサプライチェーン全体で追跡 |

**なぜ重要か：** 改善活動の「成果」を金額換算できなければ、経営に対してDX投資の正当性を示せない。コスト可視化はDX継続の燃料。

## 5. 作業の自動化（ロボット化）

自動化のステップは「自動起動技術確立 → コボット導入 → 完全自律セル構築」の3段階。

```
Step 1：自動起動技術の確立
  → 既存設備のPLC/センサ統合、自動起動条件の整備

Step 2：協働ロボット（コボット）の導入
  → 人とロボットが同一エリアで作業
  → 組立・搬送・検査補助から開始

Step 3：完全自律作業セルの構築
  → 無人運転・自律判断・異常自動停止
```

**注意点：** 「自動化できる工程か」の見極めが先。変動が大きい工程・品種切り替えが多い工程は自動化コストが高い。まず標準化。

## 6. 試運転の半自動化

製品の試運転は職人的ノウハウが集中する工程。ここをデジタル化できると大きなリードタイム短縮になる。

| ステップ | 内容 |
|---------|------|
| ノウハウの数値化 | 試運転条件・判断基準を定量パラメータで記述 |
| 計画の半自動化 | 過去実績から試運転計画をAIドラフト生成 |
| データの自動収集・分析 | センサデータのリアルタイム取得・合否自動判定 |
| 完全自動化 | 試運転工程全体の自律実行 |

## 7. 匠の技のデジタル化

熟練技能者の引退が近づく製造業において、**技能の数値化・継承**は最重要課題のひとつ。

### 対象工程の例

| 工程 | 数値化パラメータ | 自動化水準 |
|------|---------------|-----------|
| 溶接 | 電流・電圧・速度・角度・トーチ距離 | 半自動化 → 自動化 |
| バランシング | 不釣合い量・修正位置・回転数 | 自動化 |
| 精密組立 | 締付トルク・挿入力・変位量 | 半自動化 |

**進め方：**
1. 熟練者の作業を動作センサ・カメラで記録
2. 「うまくいったとき」のパラメータを機械学習
3. AI判定モデルを構築 → 設備に組み込み
4. 新人でも熟練者と同品質を実現

## 14. データ・デジタル基盤（全テーマの土台）

**ここが弱ければ何も動かない。** 最初に整備すべき領域。

### データガバナンス

| 要素 | 内容 |
|------|------|
| データ定義 | 用語・コード体系の統一（同じものを同じ名前で呼ぶ） |
| マスタ管理 | 品目・設備・人員・サプライヤのマスタを一元管理 |
| データオーナー制度 | 各データの責任者を明確化 |
| データカタログ | どこに何のデータがあるかを検索可能な状態に |
| データ品質基準 | 完全性・正確性・適時性の基準と測定の仕組み |

### デジタルインフラ

```
クラウド/エッジ基盤
├── 工場内 OT/IT ネットワーク（分離・セグメント化）
├── IoTセンサー・ゲートウェイの標準化
├── データレイク/DWH（収集・蓄積）
├── リアルタイムデータパイプライン
└── 統合データプラットフォーム
        ↓
システム連携基盤
├── ERP ↔ MES ↔ PDM の連携設計
├── APIゲートウェイ構築
└── グループ内システム横断連携
```

### サイバーセキュリティ

OT（Operation Technology）環境のセキュリティは IT 環境と異なる考え方が必要。

| 対策領域 | 内容 |
|---------|------|
| ポリシー策定 | OT/ITセキュリティポリシーの策定 |
| ネットワーク分離 | OT/IT の適切なセグメント化・アクセス制御 |
| 脆弱性管理 | 定期的なセキュリティ監査・パッチ管理 |
| インシデント対応 | 手順整備・年次訓練の制度化 |
| サプライチェーン | 調達先のセキュリティ基準への対応 |
| ゼロトラスト移行 | 「信頼しない・常に検証する」アーキテクチャへ |

## まとめ：優先度の考え方

すべてを同時に進めることはできない。以下の優先順位が実践的だ。

**最初にやること（土台）：**
1. データガバナンス・インフラ整備
2. 工作要求・技術文書の100%電子化
3. 品質記録のデジタル化

**次にやること（コア自動化）：**
4. MES導入・工程進捗のデジタル管理
5. BOP自動化（文書化が完了してから）
6. 予兆監視・設備ダッシュボード

**発展段階（AI・自律化）：**
7. AI品質予測・異常検知
8. 自律スケジューリング
9. デジタルツイン・完全自動化セル
